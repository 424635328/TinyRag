Why TinyRag

只保留真正必要的部分。

01

轻量向量化

默认使用 `bge-small-zh-v1.5`，优先尝试 CUDA，不可用时自动回退到 CPU。

02

模块化链路

文档加载、切分、检索、提示词和生成解耦，后面替换组件时不需要重写全项目。

03

双模型后端

同一套页面既能连本地 Ollama，也能切到 DeepSeek API，控制权在环境变量里。

System profile

Workflow

从文档到回答，拆成 5 个小模块。

Load

TextLoader 把知识文本统一为 Document。

Split

RecursiveCharacterTextSplitter 保留语义边界。

Embed

HuggingFaceEmbeddings 生成向量并交给 FAISS。

Retrieve

Retriever 只拿最相关片段，减少上下文开销。

Answer

LCEL 把 context、prompt 和 LLM 串成一条链。

Ask the system

把问题发给你的知识库。

页面会调用后端流式接口 `/api/chat/stream`，由 LangChain 链完成检索和回答。

运行方式

本地优先，支持热重载

默认推荐 `Ollama + qwen2.5:1.5b`。如果要走云端，只需要在 `.env` 中切换 `LLM_PROVIDER=deepseek`。`knowledge/` 下新增、修改、删除 `txt / md / markdown / pdf / csv / tsv / json / yaml / xlsx` 等文件后，下一次提问会自动重建索引。
